# PURCHASE & USER TRACKING SPECIFICATION

## 1. Identity Linking
Every action in the system flows from an authenticated identity to ensure absolute traceability.
- `user_id`: Sourced exclusively from `auth.users` via Supabase JWT.
- `ip_address`: Captured via Cloudflare headers (`CF-Connecting-IP`) at the edge during policy acceptance and checkout.

## 2. Purchase Verification
- `purchase_id`: Generated by Stripe (Checkout Session ID / Payment Intent ID).
- Supabase Webhook Handler strictly validates the Stripe Signature before inserting the `purchase_id` into `service_tickets`.
- **Idempotency**: `purchase_id` acts as a unique constraint. Re-played webhooks are discarded.

## 3. Dispute Defense (Chargebacks)
If a client issues a chargeback, the system rapidly queries:
```sql
SELECT p.ip_address, p.timestamp, p.terms_accepted, e.archive_path
FROM policy_acceptance_logs p
JOIN service_tickets s ON p.ticket_id = s.id
JOIN evidence_packs e ON s.id = e.ticket_id
WHERE s.purchase_id = 'stripe_pi_123';
```
This query returns:
1. Proof they accepted the non-refundable AI service terms.
2. The exact IP they accepted it from.
3. The cryptographic evidence pack proving the work was delivered.
